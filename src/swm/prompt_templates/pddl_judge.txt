You are an expert evaluator of whether the Evaluated plan correctly realizes the task semantics defined primarily by the Ground-truth plan.

## Inputs
- Initial image
- Instruction: {instruction}
- Ground-truth plan:
{kf_plan}
- Evaluated plan:
{nl_plan}

## Evidence Priority
Use evidence with the following priority:
1. Ground-truth plan: primary source of required task goals and logical constraints.
2. Instruction: auxiliary source for semantic clarification and referent disambiguation.
3. Initial image: auxiliary source only for object identity, spatial grounding, and clear visible contradictions.

## Decision
Evaluate PASS/FAIL. Return PASS iff: Stage 1 passes and Stage 2 passes.

## Stage 1. Goal Completeness Validation
Determine whether the Evaluated plan achieves the required end-state goals defined primarily by the Ground-truth plan.

1. Extract the minimal set of required end-state goals primarily from the Ground-truth plan.
   Use the Instruction only to clarify ambiguous wording, object identity, or references.

2. Express each required goal as an atomic, checkable final-state statement.

3. Focus on:
- target objects
- target relations or placements
- target states
- count or coverage requirements
- non-undo constraints

4. Ignore narrative or temporal markers such as “then”, “after”, “finally”, or “last”.
   Only enforce ordering when it is required by executability or task semantics.

5. Do not treat replaceable intermediate actions or nonessential execution details as goals unless they are necessary to preserve the Ground-truth plan’s task semantics.

FAIL Stage 1 if any required goal:
- is not satisfied by the end, or
- is satisfied but later undone or contradicted.

## Stage 2. Action Correctness Validation
Starting from the initial image state, check the Evaluated plan step by step.

Use the Ground-truth plan as the main semantic standard.  
Use the image only as auxiliary evidence for object grounding and clear initial contradictions.

FAIL Stage 2 if any step violates the rules below.

### 2.1 Consistency Validation
- Hand state consistency:
  If a hand is holding object o, it cannot manipulate a different object with the same hand.

- World-state consistency:
  If the initial image clearly shows a contradictory fact, FAIL.
  If something is unclear or not visible, mark it as unknown and do not invent facts.

- Ground-truth semantic consistency:
  Each step must remain logically consistent with the task semantics implied by the Ground-truth plan.
  Do not require identical wording or nonessential intermediate steps.

- Hand transfers:
  Explicit hand-switching is optional if a single-hand execution is physically feasible and preserves the same task semantics.

### 2.2 Physical Executability Validation
Verify that each step is physically executable.

Examples of invalid cases:
- placing an object without first picking it up
- washing an object without turning on the faucet
- starting an appliance without placing the object inside
- putting an object into a closed container
